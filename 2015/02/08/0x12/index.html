每日函数

发表于 2015-02-08   |   分类于 H-blog

今天心血来潮，，打算从今天开始不定时的在这篇博文中记录几个PHP或SQL好用的函数～_～，这样做一来可以慢慢建立一个我可以快速查询的字典，二来可以让我坚持学习= =

  • PHP : array_merge(arr1 ,arr2, …) // 合并数组，参数为数组，返回参数中数组合并的结果
  • SQL : sleep(int x) //数据库休息 x 秒

============write time : 8/2 2015===============

  • SQL : mysqldump -u [用户名] -p [密码] [数据库名] > [导出的文件名] //进行本地数据库导出到本地
  • SQL : source [导入文件名] //在use数据库之后,可以进行导入..
    *SQL : mysqldump -h [主机名] -u [用户名] -p [密码] [数据库名] > [导出的文件名] //把远程数据库导出到本地

==============write time : 13/2 2015 ===============

  • SQL : group_concat()
    eg : 在test数据库下有个user表,表中有个id字段,则 select group_concat(id) from user; 查询出该表下所有id的值…
  • PHP : substr(x,y,z) //此函数的功能为截取字符串, —> 从x字符串中第y+1位(这是因为第一位为0)开始截取z个字符,最后返回截取的字符…….

============write time : 15/2 2015 ===================

  • PHP : file_put_contents($a,$b) //$a -> 写入哪个文件 $b -> 写入的内容
    //最近遇到一种情况,修改已经发布的项目.这时候就不能用 var_dump() 输出变量.我自己就想了一个笨方法,,把变量输出到文件中,用到了上面这个函数.
    不过,数组变量却无法输出到文件,,,这时候又发现一个函数 : var_export($array, true)
    最后变成这样 : file_put_contents('test.txt',var_export($array, true), FILE_APPEND);
    //后面发现var_export()和var_dump()是一个类型的函数…不过我觉得dump更好用..
    最后一个参数 : FILE_APPEND 表示可以在文件最后追加内容.
  • SQL : show table status; //获取当前数据库所有表的详细信息
    //如果只想获取其中具体表的信息可以用 like '表名' 或 where Name='表名'
    eg : show table status like ‘表名’; OR show table status where Name=’表名’;

===================write time : 25/2 2015================
